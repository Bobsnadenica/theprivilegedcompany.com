# България в Данни · БвД

Статичен, локален и проверим визуален атлас на публичните отворени данни за България. Пет тематични раздела покриват всичките 14 официални категории, а пълният каталог дава достъп до всеки наличен запис. При посещение приложението работи само с публикуваната локална моментна снимка и не изтегля данни от външни услуги.

**Публикуван адрес:** [theprivilegedcompany.com/dev/bg/](https://www.theprivilegedcompany.com/dev/bg/)

## Визуално покритие

- **Транспорт** — интерактивна карта на ПТП, филтри, показатели и регионални класации.
- **Икономика** — национална серия за БВП и профил на финансовите набори.
- **Природа** — горски индикатор, земеделие, околна среда и енергетика.
- **Хора** — население, интернет, образование и здравеопазване.
- **Държава** — управление, региони, наука, правосъдие и международни отношения.
- **Каталог** — всички набори, категории, издатели, формати и локално търсене.

## Изисквания

- Node.js 20 или по-нова версия
- npm
- Python 3 само при обновяване на изходните данни

## Локално стартиране

```bash
cd dev/bg
npm ci
npm run dev
```

Vite обслужва приложението с production подпът `/dev/bg/`, затова локалният адрес е този, който Vite отпечатва, последван от `/dev/bg/`.

## Build, тест и публикуване

Само production build:

```bash
npm run build
```

Готовите междинни файлове се записват в игнорираната директория `dist/`.

Build и синхронизиране на файловете, които GitHub Pages обслужва директно от `/dev/bg/`:

```bash
npm run deploy
```

Пълна проверка на данните, subpath връзките и статичния сайт:

```bash
npm test
```

`npm run deploy` обновява само генерираните deployment файлове: `index.html`, `assets/`, `favicon.svg`, `og.png` и четирите плоски JSON файла в `data/`. Изходните файлове на приложението са отделени в `app/`, така че следващият build не използва вече генериран HTML.

За временно публикуване на друг адрес base path може да бъде променен:

```bash
SITE_BASE_PATH=/друг-път/ npm run build
```

Production стойността по подразбиране е `/dev/bg/`. Не променяйте ръчно URL адресите в генерирания `index.html`; Vite добавя правилната основа към JavaScript, CSS, иконата и всички runtime заявки за данни.

## Обновяване на моментната снимка

```bash
./update.sh
```

Скриптът:

1. изтегля всички публични български каталожни записи през официалния API;
2. разделя набори, ресурси, организации и категории в ясна структура под `data/`;
3. обновява четирите национални серии от Световната банка;
4. открива текущия ресурс на МВР, изтегля ПТП от 2024 г. насам и създава локалната карта с public-domain контур от Natural Earth;
5. проверява пълнота, идентификатори, връзки, тематични бройки, времеви серии и всяка координата на картата;
6. заменя старата моментна снимка атомично само ако няма грешки;
7. изпълнява тестовете и статичния build.

По подразбиране се използва тестовият портал. За продукционната среда:

```bash
PORTAL_BASE=https://data.egov.bg ./update.sh
```

Процесът работи само с локалните файлове на модула.

Текущият ПТП ресурс се открива автоматично. Само при диагностика може да бъде указан ръчно:

```bash
ROAD_RESOURCE_UUID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx ./update.sh
```

## Структура на проекта

```text
dev/bg/
├── app/                 # React интерфейс и Vite HTML entry
├── data/                # Канонична моментна снимка + deployment JSON
├── public/              # Статични и генерирани build входове
├── scripts/             # Обновяване, подготовка и deployment sync
├── tests/               # Качество на данните и subpath проверки
├── assets/              # Генерирани и публикувани JS/CSS bundles
├── index.html           # Генериран вход за GitHub Pages
├── package.json
└── vite.config.ts
```

`node_modules/`, `dist/`, `public/data/`, кешовете, временните директории и локалните бази са изключени в основния `.gitignore`. Каноничната снимка под `data/` и deployment файловете остават version-controlled, за да може GitHub Pages да обслужва приложението без build pipeline.
