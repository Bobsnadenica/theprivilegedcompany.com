# България в Данни · БвД

Статичен, локален и проверим визуален атлас на публичните отворени данни за България. Пет тематични раздела покриват всичките 14 официални категории, а пълният каталог дава достъп до всеки наличен запис. При посещение приложението работи само с публикуваната локална моментна снимка и не изтегля данни от външни услуги.

**Публикуван адрес:** [theprivilegedcompany.com/dev/bg/](https://www.theprivilegedcompany.com/dev/bg/)

## Визуално покритие

- **Транспорт** — интерактивна карта на ПТП, филтри, показатели и регионални класации.
- **Икономика** — национална серия за БВП и профил на финансовите набори.
- **Природа** — горски индикатор, земеделие, околна среда и енергетика.
- **Хора** — население, интернет, образование и здравеопазване.
- **Държава** — управление, региони, наука, правосъдие и международни отношения.
- **Каталог** — всички набори, категории, издатели, формати и локално търсене.

## Изисквания

- Node.js 20 или по-нова версия
- npm
- Python 3 само при обновяване на изходните данни

## Локално стартиране

```bash
cd dev/bg
npm ci
npm run dev
```

Vite обслужва приложението с production подпът `/dev/bg/`, затова локалният адрес е този, който Vite отпечатва, последван от `/dev/bg/`.

## Build, тест и публикуване

Само production build:

```bash
npm run build
```

Готовите междинни файлове се записват в игнорираната директория `dist/`.

Build и синхронизиране на файловете, които GitHub Pages обслужва директно от `/dev/bg/`:

```bash
npm run deploy
```

Пълна проверка на данните, subpath връзките и статичния сайт:

```bash
npm test
```

`npm run deploy` обновява само генерираните deployment файлове: `index.html`, `assets/`, `favicon.svg`, `og.png` и четирите плоски JSON файла в `data/`. Изходните файлове на приложението са отделени в `app/`, така че следващият build не използва вече генериран HTML.

За временно публикуване на друг адрес base path може да бъде променен:

```bash
SITE_BASE_PATH=/друг-път/ npm run build
```

Production стойността по подразбиране е `/dev/bg/`. Не променяйте ръчно URL адресите в генерирания `index.html`; Vite добавя правилната основа към JavaScript, CSS, иконата и всички runtime заявки за данни.

## Обновяване на моментната снимка

```bash
./update.sh
```

Скриптът:

1. изтегля всички публични български каталожни записи през официалния API;
2. разделя набори, ресурси, организации и категории в ясна структура под `data/`;
3. обновява четирите национални серии от Световната банка;
4. открива текущия ресурс на МВР, изтегля ПТП от 2024 г. насам и създава локалната карта с public-domain контур от Natural Earth;
5. проверява пълнота, идентификатори, връзки, тематични бройки, времеви серии и всяка координата на картата;
6. заменя старата моментна снимка атомично само ако няма грешки;
7. изпълнява тестовете и статичния build.

## Източници на данни

`update.sh` използва следните външни източници. Самият публикуван сайт не прави заявки към тях — той чете само проверените локални JSON файлове под `data/`.

### 1. Български Портал за отворени данни

- [Тестов портал и каталог](https://testdata.egov.bg/data) — източникът по подразбиране за пълната каталожна моментна снимка.
- [Продукционен портал и каталог](https://data.egov.bg/data) — използва се при `PORTAL_BASE=https://data.egov.bg ./update.sh`.
- [API на тестовия портал](https://testdata.egov.bg/api) и [API на продукционния портал](https://data.egov.bg/api).
- Точни POST endpoints: [`/api/listDatasets`](https://testdata.egov.bg/api/listDatasets) за наборите и вградените им ресурси и [`/api/listOrganisations`](https://testdata.egov.bg/api/listOrganisations) за издателите.

От портала се записват каталожните метаданни за всички 14 категории: набори, ресурси, организации, формати, лицензи, връзки и дати на обновяване. Съдържанието на всеки отделен ресурс не се изтегля масово; изключение е конкретният ресурс на МВР за картата на ПТП.

### 2. Световна банка — Отворени данни

- [API за България](https://api.worldbank.org/v2/country/BGR/indicator) — общата основа за заявките. Скриптът добавя `?format=json&per_page=100` и запазва последните 12 налични години.
- [Население, общо — `SP.POP.TOTL`](https://data.worldbank.org/indicator/SP.POP.TOTL?locations=BG)
- [БВП в текущи щатски долари — `NY.GDP.MKTP.CD`](https://data.worldbank.org/indicator/NY.GDP.MKTP.CD?locations=BG)
- [Горски територии — `AG.LND.FRST.ZS`](https://data.worldbank.org/indicator/AG.LND.FRST.ZS?locations=BG)
- [Лица, използващи интернет — `IT.NET.USER.ZS`](https://data.worldbank.org/indicator/IT.NET.USER.ZS?locations=BG)

Точният API адрес за една серия е:

```text
https://api.worldbank.org/v2/country/BGR/indicator/{INDICATOR}?format=json&per_page=100
```

### 3. Министерство на вътрешните работи — ПТП

- [Набор „Актуални данни за ПТП в периода от 01.01.2024 до момента“](https://data.egov.bg/data/view/4b948dd7-c9ef-4239-b2de-9b8e1c312467) — стабилната страница на набора в продукционния портал.
- [Текущ ресурс в тази моментна снимка](https://data.egov.bg/data/resourceView/1c58bd72-9de7-472f-9ee2-7d20fb3cded5) — ресурсът може да бъде заменен от издателя при бъдещо обновяване.
- [`/api/getResourceData`](https://data.egov.bg/api/getResourceData) — POST endpoint, от който се изтегля табличното съдържание чрез открития `resource_uuid`.

`update.sh` първо отваря страницата на набора, открива актуалния UUID на ресурса и след това изтегля ПТП. `ROAD_PORTAL_BASE` избира средата, а `ROAD_RESOURCE_UUID` може временно да фиксира конкретен ресурс за диагностика.

### 4. Natural Earth — контур на България

- [Natural Earth Admin 0 Countries, 1:10m](https://www.naturalearthdata.com/downloads/10m-cultural-vectors/10m-admin-0-countries/) — описание на набора и лиценз.
- [Директен ZIP архив](https://naciscdn.org/naturalearth/10m/cultural/ne_10m_admin_0_countries.zip) — файлът, който updater-ът изтегля.

От архива се избира държавата с код `BGR`, а контурът се опростява за статичната карта. Natural Earth обозначава този набор като обществено достояние (`Public domain`).

### Произход на текущата моментна снимка

След успешно обновяване точните източници и параметри остават записани локално:

- [`data/manifest.json`](data/manifest.json) — източници, UTC време на генериране, размери и SHA-256 хешове;
- [`data/catalog/portal-summary.json`](data/catalog/portal-summary.json) — избран портал, API адрес и контролни бройки;
- [`data/indicators/world-bank.json`](data/indicators/world-bank.json) — кодове на сериите, дати и изходни връзки;
- [`data/visuals/road.json`](data/visuals/road.json) — UUID на набора и ресурса на МВР, API адрес и версия на Natural Earth;
- [`data/validation/report.json`](data/validation/report.json) — резултатите от проверките преди активиране на snapshot-а.

По подразбиране се използва тестовият портал. За продукционната среда:

```bash
PORTAL_BASE=https://data.egov.bg ./update.sh
```

Процесът работи само с локалните файлове на модула.

Текущият ПТП ресурс се открива автоматично. Само при диагностика може да бъде указан ръчно:

```bash
ROAD_RESOURCE_UUID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx ./update.sh
```

Виж [data/README.md](data/README.md) за структурата, обхвата на проверките и разликата между проверени каталожни метаданни и проверено съдържание на отделен ресурс.

## Структура на проекта

```text
dev/bg/
├── app/                 # React интерфейс и Vite HTML entry
├── data/                # Канонична моментна снимка + deployment JSON
├── public/              # Статични и генерирани build входове
├── scripts/             # Обновяване, подготовка и deployment sync
├── tests/               # Качество на данните и subpath проверки
├── assets/              # Генерирани и публикувани JS/CSS bundles
├── index.html           # Генериран вход за GitHub Pages
├── package.json
└── vite.config.ts
```

`node_modules/`, `dist/`, `public/data/`, кешовете, временните директории и локалните бази са изключени в основния `.gitignore`. Каноничната снимка под `data/` и deployment файловете остават version-controlled, за да може GitHub Pages да обслужва приложението без build pipeline.
